Chrome Platform Security: Defense in Depth
 
Chromebooks use multiple layers of built-in security (hardware & software) for
always-on protection. The key design characteristics of each Chrome OS device
enforce fundamental principles of how data is stored, encrypted and the list of system
level services running; these include:
  • Having a read-only system partition,
  • Redundant duplicate images,
  • Automatic system verification at every boot.

Furthermore, no privilege can be granted, to a user or application, that would allow changes to system services. This fundamentally reduces the surface vectors available for attacks of the platform to almost zero.

Seamless OS Updates
 
Regular “behind the scene” automatic updates at least every six weeks aim to ensure Chrome devices have the latest security features and fixes.

When vulnerabilities are discovered, Google will roll out updates immediately to fix the issue, not depending on a planned update schedule.

“Chromebooks’ rigorous security settings ensure our clients’ data is safe. Chromebooks manage software updates automatically, so devices are always running the
latest and most secure version.” -- Charles Schwab ( Blog post)
 
Ease of Deployment and Ongoing Management
 
Manage 10s, 100s, even 1000s of Chrome OS devices with ease via
the web-based Chrome Device Management through profiles that help your IT team easily deploy and remotely control devices, and also enrich and restrict users’ experience, and apps’ behavior across a fleet of Chrome devices.Enforcing additional restrictions through Chrome Device Management can further minimize dependence on users to act diligently, such as the mandatory use of ‘Safe Browsing’, Google’s Machine Learning based algorithms to detect and block suspicious websites.

“Almost overnight, we reduced the time we spent handling IT tasks like software updates and patches, and worrying about security concerns.” -- BBB ( Blog Post)
